# Skill: LinkedIn Posts for This Project

## Purpose
Use this skill when creating LinkedIn content based on this project, including:
- feature launches
- progress updates
- lessons learned
- product decisions
- user pain points
- technical breakthroughs
- product philosophy
- behind-the-scenes posts

## Goal
Turn project work into strong LinkedIn posts that are:
- easy to understand
- engaging in the first 2 lines
- useful to the target audience
- written like a human, not like corporate AI sludge

## Audience
Primary audience:
- recruiters
- founders
- startup operators
- HR professionals
- product-minded professionals
- tech-curious business people

Secondary audience:
- general LinkedIn audience
- potential users
- potential collaborators
- potential investors

## Tone
Write in a tone that is:
- sharp
- clear
- conversational
- natural
- opinionated when useful
- not cringe
- not overly polished
- not robotic
- not corporate

Avoid:
- empty motivational phrases
- generic “thought leadership”
- obvious AI-sounding wording
- excessive emojis
- fake inspirational endings

## Writing Style
Posts should usually:
1. Start with a strong hook
2. Introduce one concrete problem, tension, or insight
3. Explain it with one clear example
4. Land on a useful takeaway, opinion, or CTA

Prefer:
- short paragraphs
- punchy lines
- real examples
- practical observations
- specific wording over vague abstractions

## Content Rules
When possible, extract content from:
- product features
- UX decisions
- bugs or product frustrations
- user behavior
- technical constraints
- launch learnings
- founder/operator lessons
- mistakes and tradeoffs

Good post angles include:
- “what we thought vs what happened”
- “a small product detail that changes everything”
- “why most teams get this wrong”
- “the hidden friction users actually feel”
- “what building this taught us”
- “what looks small but matters a lot”

## Output Preferences
When asked for LinkedIn content:
- default to giving 3 post options
- each option should have a different angle
- keep each post natural and publishable
- avoid sounding like a press release

If the user asks for only one post, give the strongest one.

## Optional Add-ons
If useful, also provide:
- 3 hook alternatives
- 1 CTA alternative
- 3 possible post titles/themes
- a shorter version for fast posting

## Hard Rule
Do not write bland, generic LinkedIn fluff.
Every post should feel like it came from someone who is actually building something.
